404CTF (Université Télécom Sud Paris / DGSE / OVHcloud) - 10 mai au 1 juin 2025

Classement : 547ᵉ sur 2 893 participants (top 20 %)


OSINT – 404CTF J’ai participé à 7 challenges OSINT dans le cadre de l’événement de cybersécurité 404CTF, organisé par l’Université Paris-Saclay, la DGSE et OVHcloud.
Ces épreuves m’ont permis d’exercer mes compétences en investigation numérique, en mobilisant des outils et méthodes variés, parmi lesquels :
- Recherche d’informations sur les réseaux sociaux
- Utilisation de SunCalc pour estimer l’heure et la date de prise de vue d’une photo
- Analyse d’articles de presse pour extraire des données contextuelles
- Consultation de rapports d’accidents aériens pour localiser un crash (challenge "Heart Attack")
- Géolocalisation avancée et recherche commerciale pour identifier le restaurant le plus éloigné de la station spatiale de Kourou et obtenir le prix d’une boisson (challenge "Addition Salée")
- Identification d’oiseaux à partir d’un enregistrement audio afin de confirmer une localisation en Guyane (flag : Lamprospiza melanoleuca) (utilisation de l'application mobile BirdNet pour l'identification d'oiseaux)
* Les write-ups détaillés sont disponibles sur demande.


Sécurité Web – 404CTF J’ai résolu plusieurs challenges portant sur la sécurité des applications web, qui m’ont permis de pratiquer l’exploitation de vulnérabilités courantes dans un environnement contrôlé.
Parmi les failles exploitées :
- JWT falsification : manipulation de jetons d’authentification JSON Web Token pour contourner les mécanismes de vérification d’identité Write Up
- Path Traversal : exploitation d’une faille de navigation dans l’arborescence du système de fichiers pour accéder à des fichiers sensibles ou protégés Write Up
Ces challenges m'ont permis de renforcer ma compréhension du fonctionnement des applications web côté serveur, des erreurs de configuration courantes et de l’impact des mauvaises pratiques de sécurité dans la gestion des données et des accès.


Analyse Forensique - 404CTF Résolution d'un challenge en analyse forensique portant sur l’exfiltration de données via un périphérique USB à partir d’un fichier de capture réseau au format .pcapng.
Étapes clés de l’analyse :
- Ouverture du fichier avec Wireshark et application d’un filtre usb pour cibler les communications USB
- Identification de paquets URB Bulk OUT contenant des données suspectes via le champ Leftover Capture Data
- Extraction des données brutes depuis ces paquets en les exportant au format binaire (.bin) pour conserver l'intégrité des octets
- Conversion du contenu binaire en texte pour révéler le flag
Write Up


Sécurité Matérielle - 404CTF Dans ce challenge, j’ai dû reconstruire un signal audio à partir d’un fichier IQ (chall.iq) contenant des données complexes issues d’une transformée de Fourier.
L’objectif était de restaurer le signal temporel d’origine afin de l’écouter et de retrouver le flag audio.
Étapes principales :
- Chargement des données IQ (nombres complexes) avec numpy
- Application de l’IFFT (Inverse Fast Fourier Transform) pour repasser du domaine fréquentiel au domaine temporel

Traitement du signal :
- Extraction de la partie réelle
- Normalisation puis conversion en int16
- Sauvegarde du signal au format .wav (44100 Hz) à l’aide de scipy.io.wavfile.write
- Écoute du fichier reconstruit pour identifier le flag audio
Ce challenge m’a permis de découvrir le traitement du signal, en particulier les concepts de FFT/IFFT.
Bien que guidé à 70% dans la démarche, j’ai pris soin de comprendre chaque étape, ce qui m’a donné une base solide pour appréhender la manipulation de signaux numériques. Write Up

TryHackMe Basic Pentesting (niveau : Facile)

- Reconnaissance : Scan Nmap pour identifier les services actifs (HTTP, SSH, SMB)
- Web Enumeration : découverte d’un dossier caché avec Gobuster, puis récupération d’infos
- Enumération SMB : enum4linux utilisé pour identifier des utilisateurs
- Brute-force SSH : Hydra utilisé avec la wordlist rockyou.txt pour trouver un mot de passe d'un utilisateur
- Connexion SSH : Pour accèder aux utilisateurs
- linPEAS : utilisé pour détecter la présence d’une clé privée SSH appartenant à un autre utilisateur , (l'outil linPEAS a montré un point d’entrée potentiel vers un autre utilisateur, en trouvant une clé SSH mal sécurisée)
- Brut Force Attack : cassé la clé de chiffrement RSA pour obtenir un hash et convertir ce hash en un mot de passe plus lisible avec JohnTheRipper
Compétences acquises :
- Nmap (scan réseau et détection de services)
- Gobuster (brute-force de répertoires web)
- enum4linux(pour l'identification des utilisateurs)
- brute-force SSH avec Hydra, - linPEAS (énumération locale avancée pour détecter des failles, fichiers sensibles ou clés SSH exploitables)
- JohnTheRipper (pour le déchiffrement)
- * l'élévation de privilèges a été possible grâce a l'outil linPeas qui a localisé une clé SSH exploitable et grâce a l'outil JohnTheRipper qui a converti la clé RSA en un hash et qui a cassé le hash dans le but d'obtenir un mot de passe lisible
